### T3: 文本分块器（Chunker）

**交付内容**：纯 Dart 文本分块工具，支持按标题层级切分，~500 Token 窗口 + 50 Token 重叠

| 步骤 | 操作 | 产出 |
|------|------|------|
| 3.1 | 实现 **Token 估算器**：基于中英文混合的字符级估算（~1 token ≈ 1.3 英文词 ≈ 1.8 中文字） | `lib/core/rag/chunker.dart` |
| 3.2 | 实现 **按标题层级切分**：识别 `##` / `###` / `====` 等 Markdown 标题语法 | 同上 |
| 3.3 | 实现 **滑动窗口切分**：对无标题长文本按 ~500 Token 窗口 + 50 Token 重叠切分 | 同上 |
| 3.4 | 定义 `Chunk` 数据模型（id, content, pageTitle, section, seqIndex） | 同上 |
| 3.5 | 提供两种策略的统一入口：`chunkByHeadings(text)` 和 `chunkBySliding(text)` | 同上 |

**设计要点**：
- Token 估算使用 Dart 纯计算（含 `_countTokens(String)` 私有方法），无需调用 LLM
- 中文按字符数 ÷ 1.8 估算，英文按空格分词后 × 1.3 估算，混合文本加权平均
- 标题层级信息保留在 `Chunk.section` 字段，用于后续检索结果展示上下文

### T7: sqlite-vec 向量存储 + 检索

**交付内容**：封装 sqlite-vec 的向量存取，提供语义搜索接口，含纯 Dart cosine similarity 回退

| 步骤 | 操作 | 产出 |
|------|------|------|
| 7.1 | 定义完整数据库 Schema（`chunks` 表 + `chunk_embeddings` 虚拟表 + `books` 表） | `lib/core/rag/vector_store.dart` |
| 7.2 | 封装 **sqlite-vec 初始化**：尝试加载 FFI 绑定，失败时标记 fallback 模式 | 同上 |
| 7.3 | 实现 **chunk 写入**：`insertChunk(Chunk, embedding)` → 同时写 chunks 表和 embeddings 表 | 同上 |
| 7.4 | 实现 **批量 chunk 写入**：`insertChunks(List<Chunk>, List<List<double>>)` → 事务批量插入 | 同上 |
| 7.5 | 实现 **语义搜索**：`search(queryEmbedding, topK, sourceType?)` → 返回 Top-K 相似段落 | 同上 |
| 7.6 | 实现 **纯 Dart cosine similarity 回退**：全量读取向量 → 内存计算余弦相似度 → 返回 Top-K | 同上 |
| 7.7 | 实现 **删除**：`deleteBySource(sourceType, sourceId)` → 清除书籍/Wiki 相关向量 | 同上 |
| 7.8 | 实现 **统计**：`getStats()` → 返回向量总数、Wiki 页面数、书籍数 | 同上 |
| 7.9 | 创建 Riverpod Provider | `lib/core/rag/vector_store_provider.dart` |

**数据库 Schema**（来自 implementation_plan.md）：

```sql
CREATE TABLE chunks (
  id          TEXT PRIMARY KEY,
  source_type TEXT NOT NULL,         -- 'wiki' | 'book'
  source_url  TEXT,                  -- Wiki 页面 URL（wiki 类型）
  wiki        TEXT,                  -- 'prts' | 'warfarin'（wiki 类型）
  book_id     TEXT,                  -- 关联 books.id（book 类型）
  page_title  TEXT,                  -- Wiki 页面标题 或 书籍章节标题
  section     TEXT,                  -- 所属小节标题
  content     TEXT NOT NULL,
  updated_at  INTEGER
);

CREATE VIRTUAL TABLE chunk_embeddings USING vec0(
  chunk_id TEXT PRIMARY KEY,
  embedding FLOAT[1536]
);

CREATE TABLE books (
  id           TEXT PRIMARY KEY,
  file_name    TEXT NOT NULL,
  display_name TEXT,
  chunk_count  INTEGER,
  imported_at  INTEGER
);
```

**设计要点**：
- sqlite-vec 的 `vec0` 虚拟表要求 Flutter FFI 支持，Android 8.0+ (API 26) 兼容性需要验证
- 回退方案：当 `sqlite_vec.load()` 抛出异常时，设置 `_useFallback = true`，后续搜索改用纯 Dart 实现
- 纯 Dart 回退：向量以 BLOB 存入普通 SQLite 表 `chunk_embeddings_fallback(chunk_id TEXT, embedding BLOB)`，搜索时全量加载到内存做 cosine similarity（数据量 < 10 万条时性能可接受）
- Top-K 结果包含 `Chunk` 完整信息 + 相似度分数

**预计新建文件**：
- `lib/core/rag/vector_store.dart`
- `lib/core/rag/vector_store_provider.dart`

**commit**: `feat(rag): implement sqlite-vec vector store with pure Dart cosine similarity fallback`

## 文档中的模糊/风险点

1. **sqlite-vec Flutter FFI 兼容性**：sqlite_vec 的 Flutter 原生绑定在 Android 8.0+ / iOS 14+ 上未完全验证。如果 FFI 不可用，纯 Dart cosine similarity 回退性能对于大量向量（>10 万条）可能较慢。建议在实现后优先在真实设备上测试。
2. **终末地 Wiki 域名**：`implementation_plan.md` 提到终末地 Wiki 但未给出确定域名。目前假设为类似 `wiki.endfield.moe` 的地址，建议在设置页中允许用户自定义 Wiki Base URL。
3. **Embedding 模型维度**：`implementation_plan.md` 指定 1536 维（text-embedding-3-small），但用户可能使用其他 Embedding 模型（如 `text-embedding-3-large` 的 3072 维）。目前固定在 1536，后续需考虑动态维度支持。
4. **`sqlite_vec` 包的状态**：该包处于早期阶段（`^0.1.x`），API 可能有变动，需要在实现时仔细阅读其当前文档。`sqflite` + `sqlite_vec` 的双数据库实例需要协调。
5. **PDF 提取的质量**：`syncfusion_flutter_pdf` 的文本提取质量取决于 PDF 的编码方式（扫描版 PDF 无法提取文字）。需要在资料页添加提示："部分扫描版 PDF 可能无法提取文字"。
