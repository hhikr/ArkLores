# v0.9 Visual Design

记录日期：2026-07-15。本文只定义视觉系统；产品与证据契约仍以
`implementation_plan.md` 为准。

## Reference Provenance

本轮直接设计输入是用户在 2026-07-15 会话中提供的两张生成式设置页草图：一张终末地
方向，一张明日方舟方向。它们用于确认颜色关系、密度和构图意图，不是游戏实机截图，
也不作为官方 UI 的事实证据。

从草图提取的可复用规律：

| 方向 | 采用的规律 | 不直接复制的内容 |
| --- | --- | --- |
| 明日方舟 | 中性黑与炭灰工业底色、蓝色导视、档案式层级、细线和切角 | 角色立绘、罗德岛 Logo、游戏字标、游戏截图 |
| 终末地 | 大面积白色、少量浅灰层级、亮黄色重点色、克制高密度、设备面板式边框 | 角色立绘、终末地 Logo、舰船图、游戏字标 |

最终 App 不打包草图或游戏资产。品牌感由代码绘制的线、网格、切角、色标和排版层级
表达，Material Icons 只承担通用操作语义。

## Shared Information Architecture

两套主题必须保持相同的页面顺序和操作位置，避免切换主题后重新学习：

```text
page header
  theme and language preferences
section header: AI service
  API settings
section header: knowledge base
  GameData management
section header: help
  onboarding
bottom navigation
```

差异仅来自视觉语法：明日方舟使用黑灰表面和蓝色左侧导视；终末地使用白色主表面、
少量浅灰层级、黄色重点色和双端状态标记。

## Token Decisions

- 背景分为 page、navigation、card、elevated control 四级，不再用一种蓝黑色覆盖所有层级。
- 夜间主题重点色为 `#0BA0D0`，背景为中性近黑与炭灰；蓝色只用于当前状态和导视。
- 日间主题重点色为 `#F8D439`，背景以柔和白色为主、浅灰为辅；黄色作为色块、色条和
  边框承载重点，浅色表面上的文字与线性图标使用深灰以保持对比度。
- 面向用户只显示“日间 / 夜间”，与两款游戏相似的视觉风格作为彩蛋，不直接使用游戏名。
- 正文和标题统一使用 Noto Sans SC，靠字重、字号和空间建立层级，不用装饰字体牺牲中文一致性。
- 卡片采用 10-12 px 切角、1 px 边框和局部色标；不使用大圆角、玻璃拟态或持续发光。
- 导航高度固定为 68 px（另加安全区），动态标签不得改变控件尺寸。
- 页面背景使用透明度不一的主次灰线、局部断线和透视汇聚线建立轻微纵深；右上角使用
  两条主题色斜线标识当前日间/夜间主题。纹理由 App 根节点统一绘制，业务页面不得复制。

## Responsive Rules

- 页面内容最大宽度 720 px，手机两侧 20 px 留白。
- 设置卡片统一为内容区宽度的 94%，水平居中且左右 margin 相同。
- 偏好项在可用宽度小于 430 px 或文字放大时转为上下布局。
- 标题和说明允许有限换行并使用省略，不允许挤压右侧操作控件。
- 底部导航图标、标签和选中线尺寸固定；每项等宽。
- v0.9 截图矩阵至少覆盖 360x800 与 420x900 logical pixels、中文/英文、双主题、
  1.0/1.6 文字缩放。

## Component Ownership

- `AppThemeTokens`：语义颜色、层级表面、文字、切角和主题身份。
- `ThemeAwareCard`：通用切角交互表面及主题特有的局部边框标记。
- `IndustrialBackdrop`：低对比代码绘制网格，不承载信息或点击行为。
- `IndustrialPageHeader` / `IndustrialSectionHeader`：页面和分区层级。
- `MainShell`：稳定尺寸的全局导航，不由业务页面重复实现。

后续 Wiki、AI、证据和知识库重做应复用这些组件；只有信息结构不同才新增业务组件，
不能复制一套硬编码颜色和边框。
