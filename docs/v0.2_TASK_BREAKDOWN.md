### T4: Wiki 夜间模式 CSS 注入

**交付内容**：根据当前 App 主题向 WebView 注入深色模式 CSS，主题切换时实时更新

| 步骤 | 操作 | 产出 |
|------|------|------|
| 4.1 | 创建 `WikiDarkMode` 工具类，生成 CSS 注入脚本 | `lib/features/wiki/wiki_dark_mode.dart` |
| 4.2 | 编写深色 CSS 覆盖规则（背景变暗、文字反色、表格适配） | CSS 字符串 |
| 4.3 | 在 WebView `onLoadStop` 时执行 `controller.evaluateJavascript` 注入 CSS | 页面加载后自动应用 |
| 4.4 | 监听 App 主题切换（通过 `ref.watch(themeProvider)`），主题变化时重新注入 | 实时跟随主题切换 |

**CSS 注入策略**：
- 使用 `evaluateJavascript` 动态注入 `<style>` 标签到页面 `<head>`
- 注入的 CSS 使用 `!important` 提高优先级，避免被页面内联样式覆盖
- 深色模式主要规则：
  - `html { filter: invert(0.85) hue-rotate(180deg); }`（基础反色）
  - `img, video, iframe { filter: invert(1) hue-rotate(180deg); }`（图片还原）
  - 额外覆盖 Wiki 特定表格和代码块样式

**预计新建文件**：`lib/features/wiki/wiki_dark_mode.dart`

**commit**: `feat(wiki): implement Wiki dark mode with CSS injection synced to app theme`

## 文档中的模糊/风险点

1. **终末地 Wiki 的 URL**：implementation_plan.md 中未明确终末地 Wiki 的具体域名。当前暂定为 `https://wiki.endfield.moe`，建议实际开发时确认真实地址（可能在设置页中配置）。
2. **iOS WebView 兼容性**：`flutter_inappwebview` 在 iOS 14 上的功能完整性需要测试，特别是 CSS 注入和 JavaScript Bridge 在 iOS WKWebView 中的表现。
3. **PRTS Wiki 反爬策略**：部分页面可能依赖 JavaScript，需确认 `InAppWebView` 默认配置即可正常渲染，无需额外设置。
4. **CSS 注入对 Wiki 布局的破坏风险**：深色 CSS 注入可能影响特定页面布局（如表格、技能图标），需在多个典型页面上验证。
