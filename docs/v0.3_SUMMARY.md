## Seed 知识库

v0.3 最终 seed bundle 分为 Flutter assets 和 GitHub Release asset：

| 文件 | 作用 |
|------|------|
| `arklores_knowledge.db.gz` | GitHub Release asset，预构建 chunks、embeddings、seed metadata |
| `wiki_cache.zip` | 对应的 Markdown Wiki 缓存 |
| `seed_manifest.json` | seed version、source 统计、embedding metadata、DB asset URL 与 SHA256 |

当前 seed DB 校验结果：

```text
PRAGMA user_version = 1
chunks = 17087
chunk_embeddings = 17087
embedding_status = ok: 17087
PRTS = 2123 distinct page titles / 15218 chunks
Endfield = 662 distinct page titles / 1869 chunks
profile_id = builtin:builtin-embedding
```

构建入口：

```bash
/opt/miniconda3/bin/python tool/build_seed.py \
  --sources=prts,endfield \
  --output=build/seeds \
  --force
```

断点续跑使用 `--resume`，不要和 `--force` 同时使用。

## 关键架构决策

### SQLite + 纯 Dart cosine similarity

`sqlite_vec` Flutter FFI 路径因当前包构建不完整暂缓。v0.3 使用 `sqflite` 保存 chunk metadata 和 embedding BLOB，检索时用纯 Dart cosine similarity 计算 Top-K。

### Embedding profile 隔离

索引数据按 `profile_id` 隔离。切换 API provider/model 或切换到内置模型不会自动删除旧索引；删除 profile 时才会同步删除该 profile 的 chunks、embeddings 和 books。API Key 不参与 profile 身份识别。

### 内置固定 embedding

内置模型固定随包发布，不提供用户替换模型能力。当前模型输出 512 维，供 seed 构建、移动端 fallback 和无 Embedding API 用户使用。

### 桌面端 seed 构建

移动端首次建库耗时过长，因此 v0.3 增加桌面端 seed pipeline：Dart 负责爬取/清洗/分块/写 DB，Python TFLite runner 负责批量生成 embedding，最终将压缩 DB 作为 release asset 发布，manifest/cache 保留在 `assets/seeds/`。

### Wiki 同步策略

预构建 seed 页面在本地已存在且无失败向量时直接跳过。PRTS 不再使用 MediaWiki `touched` 字段触发重建，因为该字段会因缓存/模板刷新产生噪声。Warfarin 同步也先检查本地 metadata，避免逐条下载健康 seed 页面。
