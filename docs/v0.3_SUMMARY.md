## 架构决策记录

### 1. 双数据库架构
- **sqflite** 继续用于书签（已有功能，不改变）
- **sqlite3 + sqlite_vec** 用于知识库向量存储（FFI 路径）
- **sqflite fallback** 当 FFI 不可用时，存储 chunk 元数据 + BLOB 向量
- 原因：`sqlite_vec` 使用 `package:sqlite3` 的 FFI 扩展加载，无法与 `sqflite` 共享连接

### 2. sqlite-vec 回退策略
- 尝试加载 `sqlite3.ensureExtensionLoaded(sqlite_vec.vec0)` 
- 成功 → 使用 `vec0` 虚拟表进行原生向量搜索
- 失败 → 降级到 sqflite + 纯 Dart 内存 cosine similarity
- 回退性能：10 万条向量以内可接受（全量加载到内存）

### 3. Embedding 维度固定 1536
- 对应 `text-embedding-3-small` 的默认维度
- 零向量处理：空文本返回全零向量，不浪费 API 配额

### 4. PDF 文本提取使用 PdfTextExtractor
- Syncfusion 的 API 为 `PdfTextExtractor(document).extractText()`
- 不是 `page.text` 属性，已使用正确 API

### 5. 引导流程独立于 MaterialApp
- OnboardingPage 在 `main.dart` 中作为 `MaterialApp.home` 的条件分支
- 配置完成后设置 flag，下次启动直接进 MainShell

1. **缺少 Agent 核心文件**：`core/agent/react_loop.dart` 和 `tools/` 下的工具文件尚未创建（v0.4 范围）
2. **终末地 Wiki URL 待确认**：当前使用 `https://wiki.endfield.moe/api.php`，需在设置页添加可配置性
3. **sqlite-vec 在 iOS 上未验证**：FFI 绑定在 iOS 14+ 上需要真机测试
4. **知识库爬取分类硬编码**：`knowledge_base_page.dart` 中 `_prtsCategories` 和 `_endfieldCategories` 是硬编码列表，后续应在设置页中可配置
