## 架构决策记录

### 1. 双数据库架构
- **sqflite** 继续用于书签（已有功能，不改变）
- **sqlite3 + sqlite_vec** 用于知识库向量存储（FFI 路径）
- **sqflite fallback** 当 FFI 不可用时，存储 chunk 元数据 + BLOB 向量
- 原因：`sqlite_vec` 使用 `package:sqlite3` 的 FFI 扩展加载，无法与 `sqflite` 共享连接

### 2. sqlite-vec 回退策略
- 尝试加载 `sqlite3.ensureExtensionLoaded(sqlite_vec.vec0)` 
- 成功 → 使用 `vec0` 虚拟表进行原生向量搜索
- 失败 → 降级到 sqflite + 纯 Dart 内存 cosine similarity
- 回退性能：10 万条向量以内可接受（全量加载到内存）

### 3. Embedding 维度动态检测（2026-07-12 追加）
- 原设计：硬编码 `1536` 维（text-embedding-3-small）
- 实测 DeepSeek 的 `deepseek-embed` 模型返回 **2048 维**，不兼容
- 改为 `Embedder` 层从首次 API 响应中自动检测维度并缓存
- 零向量使用已检测维度（未检测前默认 1536）
- 现在兼容任意 Embedding 模型：OpenAI（1536）、DeepSeek（2048）等
- 调用链：`Embedder.embed()` → API 响应 → 检测 `vector.length` → 缓存到 `_dimension`

### 4. PDF 文本提取使用 PdfTextExtractor
- Syncfusion 的 API 为 `PdfTextExtractor(document).extractText()`
- 不是 `page.text` 属性，已使用正确 API

### 5. 引导流程独立于 MaterialApp
- OnboardingPage 在 `main.dart` 中作为 `MaterialApp.home` 的条件分支
- 配置完成后设置 flag，下次启动直接进 MainShell

## 追加变更：多语言本地化（2026-07-12）

### 实现方案
使用 Flutter 官方 `flutter_localizations` + `intl` + ARB 文件方案，配合 Riverpod 控制语言切换。

### 架构
- ARB 文件位于 `lib/shared/l10n/`：`app_en.arb`（英文）+ `app_zh.arb`（中文），key 完全平行
- `flutter gen-l10n` 编译生成类型安全的 `AppLocalizations` 类（`lib/shared/l10n/generated/`）
- `context.t.xxx` 便利扩展方法（`lib/shared/l10n/l10n.dart`）
- `SupportedLocale` 枚举 + `localeProvider`（Riverpod）控制当前语言
- `MaterialApp` watch `localeProvider`，切换即时生效无需重启
- 设置页「Language」切换条（EN / 中文），`SegmentedButton` 控制 `localeProvider`

### 新增文件
| 文件 | 作用 |
|------|------|
| `lib/shared/l10n/app_en.arb` | 英文翻译（约 80+ keys） |
| `lib/shared/l10n/app_zh.arb` | 中文翻译（与 en 完全平行） |
| `lib/shared/l10n/locale_provider.dart` | 语言状态 + 切换 |
| `lib/shared/l10n/l10n.dart` | context.t 便利扩展 |
| `lib/shared/l10n/generated/app_localizations.dart` | 自动生成的类型安全类 |
| `l10n.yaml` | gen-l10n 配置 |

### 修改文件
| 文件 | 变更 |
|------|------|
| `pubspec.yaml` | 加 `flutter_localizations` + `intl` + `generate: true` |
| `lib/main.dart` | MaterialApp 添加 `locale` / `supportedLocales` / `localizationsDelegates` |
| 全部 UI 文件（11个） | 硬编码字符串 → `context.t.xxx` |
| `lib/app.dart` | BottomNavigationBar labels 改为 localized |

### 设计原则
- 中文和英文在代码结构中**完全平行**：都是 ARB 文件、都是编译时检查
- 加新语言只需新增 `app_xx.arb` + `SupportedLocale` 枚举值
- key 拼错或漏翻译 → **编译报错**，不会运行时才暴露

### 追加变更：Embedding Profile 隔离机制（2026-07-13）

用户确认旧 profile 应保留，切换 API provider/model 或切换到内置模型时不自动删除旧数据；如果用户换回旧 API/模型，可以切回旧 profile。

#### 当前实现

| 文件 | 作用 |
|------|------|
| `lib/core/llm/embedding_profile.dart` | profile 模型、backend 枚举、active profile 状态 |
| `lib/core/rag/embedding_client.dart` | API / Built-in embedding 统一接口 |
| `lib/core/rag/api_embedding_client.dart` | API embedding 后端适配器 |
| `lib/shared/providers/settings_provider.dart` | profile 列表、激活、删除状态管理 |
| `lib/features/settings/settings_service.dart` | profile 持久化与 legacy 迁移 |
| `lib/core/rag/vector_store.dart` | `profile_id` schema、profile 范围查询、profile 数据删除 |
| `lib/core/rag/wiki_indexing_provider.dart` | Wiki 索引和失败重试绑定 active profile |
| `lib/features/materials/book_import_service.dart` | 书籍导入绑定 active profile |
| `lib/features/settings/api_settings_page.dart` | profile 列表、激活、删除 UI |
| `lib/features/settings/knowledge_base_page.dart` | 当前 active profile 展示 |

#### 行为规则

- API Key 不参与 profile 身份识别；换 Key 不会创建新 profile。
- profile 身份当前按 backend + baseUrl + model 匹配，dimension 作为元数据保留。
- API profile 的 dimension 会在首次成功生成 embedding 后写回 active profile。
- 旧索引数据迁移到 `legacy` profile，升级后仍可查询。
- 当前统计、书籍列表、Wiki 索引、失败重试只作用于 active profile。
- 用户删除 profile 时，同步删除该 profile 的 chunks、embeddings 和 books。
