## 文档模糊/矛盾之处

本次开发中发现以下需确认的点：

1. **工具栏深色模式 vs App 主题的关系**：当前实现中，工具栏的深色模式按钮独立于 App 主题开关。两者初始同步（默认为 dark），但用户可通过工具栏独立控制 WebView 深色模式。这与文档「与 App 主题联动」的描述一致。

2. **书签页面路由策略**：书签管理页使用 `Navigator.push` 全屏路由（而非底部 Tab 内嵌），因为需要返回 Bookmark 对象供 WikiBrowserPage 处理跳转。这与目录结构中 `bookmark_page.dart` 的定位一致。

3. **测试策略调整**：由于 flutter_inappwebview 和 sqflite 依赖原生平台通道，无法在纯 widget 测试中渲染。将原有的 smoke test 替换为 Bookmark 模型的纯 Dart 单元测试。完整的 UI 测试需使用集成测试（integration_test）。
