## ❌ 未通过项

### P2-1：`ArkLoresApp` 中 `theme` 和 `darkTheme` 均设为 `ThemeData.dark()`

- **文件**：`lib/main.dart` 第 28-32 行
- **问题**：
  ```dart
  darkTheme: ThemeData.dark(useMaterial3: true).copyWith(...),
  theme: ThemeData.dark(useMaterial3: true).copyWith(...),   // ← light theme 也用 dark
  ```
  `theme` 参数应配置为浅色主题（`ThemeData.light()`），但当前与 `darkTheme` 同为 `ThemeData.dark()`。虽然当前 `themeMode: ThemeMode.dark` 固定为深色，不会触发此分支，但语义上不正确。
- **影响级别**：P2（不影响功能，仅代码整洁性）
- **建议修复**：
  ```dart
  theme: ThemeData.light(useMaterial3: true).copyWith(
    scaffoldBackgroundColor: theme.bgPrimary,  // 或移除，因为 themeMode 固定为 dark
  ),
  ```
  或直接移除 `theme:` 参数（因 `themeMode: ThemeMode.dark` 永远不读取它）。

### P2-2：`AnimatedSwitcher` key 将 Tab 切换与主题切换耦合

- **文件**：`lib/app.dart` 第 47-48 行
- **问题**：
  ```dart
  key: ValueKey('${theme.themeName}_$_currentIndex'),
  ```
  当用户切换 Tab 时，`_currentIndex` 改变，触发了 `AnimatedSwitcher` 的 300ms fade 动画。底部导航的 Tab 切换通常不应有 300ms 渐入渐出过渡——用户期望即时切换或更快的滑动效果。
- **影响级别**：P2（视觉体验，非功能性缺陷）
- **建议修复**：将 key 改为仅绑定主题名，在 Tab 切换时不触发 fade 动画：
  ```dart
  key: ValueKey('page_${theme.themeName}'),
  ```
  然后在 `IndexedStack` 上通过 `index` 切换保持即时响应。如需 Tab 切换动画，应使用单独的（非 AnimatedSwitcher）转场方案。

### P2-3：Widget Test 覆盖率过低

- **文件**：`test/widget_test.dart`
- **问题**：只有一个 smoke test 验证底部导航 Tab 文字渲染。缺乏对以下关键路径的测试：
  - 主题切换（`ThemeNotifier.toggle()` 状态变更）
  - `ThemeAwareCard` 在不同主题下的 Token 读取
  - `AnimatedSwitcher` 动画触发条件
- **影响级别**：P2（非阻塞，但会增加后续迭代的回退风险）
- **建议修复**：添加 3-5 个单元测试覆盖主题逻辑（纯 Dart 层，无需 widget 渲染）
