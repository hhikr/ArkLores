`flutter analyze` 无 error，但返回非零，因为当前仓库仍有既有 info/warning 级 lint，例如：

- `avoid_print`：启动和工具脚本中的调试输出
- `unnecessary_brace_in_string_interps`
- Warfarin crawler 中未使用字段 / dead null-aware expression

这些问题不阻塞 v0.3 功能发布，但建议在 v0.4 前或专门的 cleanup 分支中处理。

## 已知限制

- sqlite-vec FFI 路径仍暂缓，当前仅使用 SQLite + 纯 Dart cosine similarity。
- Endfield seed manifest 的 processed item count 与 SQLite distinct `page_title` count 不完全相同，因为当前 seed 使用显示名作为 key，重复显示名会合并到同一 `page_title`。
- `flutter analyze` 仍有 info/warning 级 lint，未作为 v0.3 发布阻塞项。
- iOS 真机未在本轮验证。
