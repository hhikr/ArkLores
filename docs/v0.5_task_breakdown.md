# v0.5 Task Breakdown

更新日期：2026-07-15

## 范围

本迭代实现 GameData-only 事实核查模式。唯一 Agent 工具是 `search_local_lore`；Wiki、
Book 和用户文本不进入证据源。不包含角色扮演、Wiki 选中文字转交、GameData schema /
builder 修改、发布 tag、Release 或 push。

## 已完成

- [x] 新增 Fact-Check Agent：拆解主张，分别检索实体、支持证据与反证，最多五轮 ReAct。
- [x] 结论使用 supported、refuted、uncertain、unavailable 四态协议。
- [x] supported / refuted 必须存在实际 GameData Result observation，否则强制降级。
- [x] prompt 区分直接证据、间接证据和证据缺失，不把 retrieval confidence 当事实确定性。
- [x] 多轮历史保留原主张与回答；话题切换必须显式说明并隔离旧证据。
- [x] UI 提供本地化四态标识、可展开 observation 引用、空状态、错误状态、取消和重试。
- [x] 单元测试覆盖工具限制、来源声明、无结果、实体歧义、结论解析和上下文追问。
- [x] 复用 Summary/ReAct 的 debug `AgentLogger`，日志标识 FactCheck，并在来源 guard 与
  verdict 校验后记录实际生效的最终结论；错误早退也会 flush。
- [x] Widget 测试覆盖窄屏、2 倍文字缩放和证据展开。
- [x] 删除 Warfarin crawler 的公网观测/正文打印测试，改为离线 parser 与 formatter 契约；
  bookmark model 测试按职责重命名。
- [x] Android 安装脚本与指南同步到 API 36；默认保留 App 数据，外部 GameData URL 默认
  强制 SHA256，并明确区分本地 release-mode 验收包与正式签名包。
- [x] GameData schema v2 增加通用 story scope 元数据；Fact-check evidence 模式按稳定
  scope ID、entity ID 与关系词交集检索剧情，普通档案和无结果不能放行确定 verdict。
- [x] 修复 provider 附加 Action metadata 时的工具名解析，以及 `handbook` 触发 Book
  source warning 的误报；固定完整 DB QA 强制使用绝对 DB 路径。
