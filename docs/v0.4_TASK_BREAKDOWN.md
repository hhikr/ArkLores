# ArkLores v0.4 — 梗概生成 Agent（首个 AI 功能）· 任务分解计划

> 本文档用于规划 v0.4 版本的开发任务。按照项目的迭代规范，所有任务均被细化为独立的子任务，以便小步提交与持续验证。

---

## 任务分解清单

### 第一阶段：ReAct Loop 核心及 Tool 机制建立
- [ ] **T1.1: 建立 Tool 基础抽象与注册表 (Tool Registry)**
  - 创建 `lib/core/agent/tools/agent_tool.dart`，定义 `AgentTool` 抽象类（包含名称、描述、参数 Schema 以及 `execute` 执行接口）。
  - 创建 `lib/core/agent/tools/tool_registry.dart`，负责统一注册与获取可供 Agent 调用的工具。
- [ ] **T1.2: 实现 `search_wiki` 工具**
  - 创建 `lib/core/agent/tools/search_wiki.dart`，继承 `AgentTool`。
  - 调用 `VectorStore` 实现语义搜索，输入关键字，返回检索到的 Top-K 文本片段及关联元数据（chunk_id、source_type、source_url 等）。
- [ ] **T1.3: 实现 `cite_source` 工具**
  - 创建 `lib/core/agent/tools/cite_source.dart`，继承 `AgentTool`。
  - 输入 `chunk_id`，从 `VectorStore` 中获取对应的完整原文内容及来源信息，便于 Agent 生成准确引用。
- [ ] **T1.4: 编写纯 Dart ReAct Loop 执行器 (ReAct Loop Core)**
  - 创建 `lib/core/agent/react_loop.dart`。
  - 设计 ReAct 执行逻辑：接受 System Prompt、用户 Query 和工具集，在多轮循环中让 LLM 生成 `Thought`（思考过程）和 `Action`/`Action Input`（工具调用），调用 Dart 侧 Tool 得到 `Observation`（观察结果），并反馈给 LLM，直至输出最终的 `Final Answer`。
  - 提供流式输出支持（流式输出最后的 Final Answer 文本，同时收集思考与工具调用过程）。

### 第二阶段：梗概生成 Agent (Summary Agent) 业务逻辑
- [ ] **T2.1: 实现 Summary Agent 核心流程**
  - 创建 `lib/core/agent/summary_agent.dart`。
  - 结合 `agent_prompts.dart` 中的 `summaryInstructions`，实现实体识别与分类引导逻辑。
  - 封装 ReAct Loop，使 Summary Agent 能够自主决定使用 `search_wiki` 与 `cite_source` 收集并生成层级摘要。
  - 创建 `lib/core/agent/agent_provider.dart`，为 UI 提供 Summary Agent 实例及其状态绑定。

### 第三阶段：AI 对话界面骨架与交互集成
- [ ] **T3.1: 升级 AI 对话页架构 (三模式 TabBar)**
  - 重写 `lib/features/ai/ai_chat_page.dart`。
  - 添加顶部切换 Tab（或 SegmentedButton）：事实核查 / 梗概生成 / 角色扮演。
  - 维护各模式独立的对话历史状态（通过 Riverpod StateProvider / Notifier）。
- [ ] **T3.2: 编写对话气泡与 Markdown 渲染组件**
  - 创建 `lib/features/ai/widgets/chat_bubble.dart`。
  - 支持用户/AI气泡区分，AI 侧使用 `flutter_markdown` 渲染。
  - 注入自定义的 Markdown 标签解析，使得文本中类似 `[Wiki:xxx]` 或 `[Book:xxx]` 的引用标记能够显示为带下划线的高亮可点击文字。
- [ ] **T3.3: 引用卡片集成与动效展示**
  - 在 `lib/features/ai/widgets/` 中实现引用展示逻辑：点击 Markdown 中的引用高亮词后，在消息下方或悬浮卡片中展示已有的 `CitationCard`。
  - 连通 `CitationCard` 的 Wiki 和 Book 颜色差异及展开/收起弹性动画。

### 第四阶段：测试、验证与文档整理
- [ ] **T4.1: Agent 逻辑及 Tool 单元测试**
  - 编写单元测试验证 ReAct Loop 执行器能够正确解析 Thought/Action，并正确调用 mock 的 Tools。
  - 测试 `search_wiki` 和 `cite_source` 能够正确从 Mocked VectorStore 获取结果。
- [ ] **T4.2: 梗概生成端到端（E2E）真机/模拟器联调与精修**
  - 在设置页配置 API Key，导入测试资料或同步 Wiki。
  - 输入典型词条（如「凯尔希」），测试 Summary Agent 能否在多轮对话中正确产出结构化的剧情梗概。
- [ ] **T4.3: 补充多语言翻译条目并撰写总结**
  - 在 `app_en.arb` 和 `app_zh.arb` 中添加本阶段 UI 涉及到的新文案（如 Tab 标题、加载状态、无建议提示等）。
  - 在 `docs/` 目录下添加 `v0.4_SUMMARY.md`。

## 交付规范要求

* **状态管理**：全部使用 Riverpod，页面无状态由 Provider 驱动。
* **主题自适应**：所有 UI 组件（包括对话气泡、加载动画、TabBar、引用卡片外层）颜色、字体和间距必须使用 `ref.watch(themeProvider)` 提供的 Token，不可硬编码。
* **Git 规范**：小步提交，按任务清单 `T1.1` 至 `T4.3` 粒度进行 commit，确保历史清晰。
