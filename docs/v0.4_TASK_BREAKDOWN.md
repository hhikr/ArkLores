# ArkLores v0.4 — Agent 基础设施 + 可替换知识源抽象 + Summary MVP · 任务分解计划

> v0.4 已从“基于 Wiki RAG 的梗概生成 Agent”调整为“Agent 基础设施 + 可替换知识源架构 + Summary MVP”。v0.3 Wiki seed DB 仅作为临时数据源，v0.4.5 将以中文 GameData release asset 重建主知识库。

---

## 目标

1. 稳定 ReAct Loop、Tool 调用、日志、错误处理和 AI 对话 UI。
2. 将 Agent 工具层从 `search_wiki` 单一数据源中解耦，为 GameData 主知识源预留接口。
3. 交付可用但不追求最终内容质量的 Summary MVP。
4. 修复空回答、回答截断无提示、非严格 `Action Input` 解析失败等基础体验问题。

---

## 第一阶段：ReAct Loop 稳定性

- [ ] **T1.1: 支持非严格 Action Input 解析**
  - 支持 `{query: 缪因, top_k: 5}`、`query: 缪因, top_k: 5` 等常见 LLM 非 JSON 输出。
  - 保留严格 JSON 优先解析。
  - 为解析失败添加日志和单元测试。

- [ ] **T1.2: 强化 ReAct Prompt 格式约束**
  - 明确 `Action Input` 必须是 strict JSON。
  - 明确最终阶段必须输出 `Final Answer:`，不得继续调用工具。

- [ ] **T1.3: 修复空最终回答**
  - `complete` 时若 assistant content 为空，显示明确错误文案。
  - Agent 日志记录 `EMPTY_FINAL_ANSWER`。

- [ ] **T1.4: 处理回答截断**
  - `LLMClient` 读取并暴露 `finish_reason`。
  - `finish_reason == length` 时触发一次续写或给 UI 明确截断提示。
  - 最终回答生成阶段提高 `maxTokens`，并优先使用 `chatStream()`。

---

## 第二阶段：Tool 层知识源抽象

- [ ] **T2.1: 定义 source-neutral Tool 返回结构**
  - Tool observation 与 debug diagnostics 分离。
  - 返回项必须包含 `source_type`、`retrieval_type`、`source_path/source_url`、`chunk_id`。

- [ ] **T2.2: 引入 `search_local_lore` 接口**
  - 当前可代理 v0.3 本地 DB。
  - 命名和参数面向未来 GameData，不再绑定 Wiki。
  - 使用 hybrid retrieval：标题精确匹配 / 标题模糊匹配 / 内容关键词 / 向量扩展。

- [ ] **T2.3: 保留 `search_wiki` 作为补充工具**
  - 语义上限定为指定 Wiki 的在线补充搜索。
  - 不作为 Summary Agent 默认主检索工具。

- [ ] **T2.4: 升级 `cite_source` 多来源兼容**
  - 支持 GameData / Wiki / Book 三类引用。
  - 当前 v0.3 DB 无 GameData 时走兼容路径。

---

## 第三阶段：Summary Agent MVP

- [ ] **T3.1: 调整 Summary Agent Workflow**
  - 实体识别 → `search_local_lore` → 必要时 `search_wiki` → `cite_source` → Markdown 输出。
  - 检索不到时必须说明限制，不得编造。

- [ ] **T3.2: 更新 Summary Agent Prompt**
  - 信任策略改为 GameData > Wiki > Book。
  - Wiki 仅为补充，Book 必须标注不确定性。
  - 若来源不足，输出“当前知识库未覆盖”。

- [ ] **T3.3: AI 对话 UI 骨架**
  - 三模式 Tab 保留。
  - Summary 页支持流式最终回答、ReAct 步骤折叠、错误状态和清空历史。

---

## 第四阶段：引用系统多来源适配

- [ ] **T4.1: 扩展引用卡 source type**
  - GameData：显示 `GameData · 角色档案 / 剧情 / 语音 / 物品`。
  - Wiki：显示站点和 URL。
  - Book：显示书名和免责声明。

- [ ] **T4.2: 引用解析兼容旧 UUID 标记**
  - 保留 `[chunk_id]` 解析。
  - 为未来 `source_type:chunk_id` 或结构化引用格式预留。

---

## 第五阶段：检索健康与测试

- [ ] **T5.1: 固定查询回归测试**
  - `阿米娅`、`凯尔希`、`罗德岛`、`龙门`、`莱茵生命`。
  - v0.3 临时 DB 阶段至少验证标题/关键词优先级和低信息过滤。

- [ ] **T5.2: built-in embedding 诊断入口**
  - 保留诊断测试。
  - Android / 可用 TFLite runtime 环境下输出不同文本 cosine。

- [ ] **T5.3: Agent 单元测试**
  - 非严格 Action Input 解析。
  - 空回答保护。
  - 截断 finish_reason 处理。
  - hybrid retrieval 排序。

---

## 交付规范要求

- **状态管理**：全部使用 Riverpod，页面状态由 Provider / Notifier 驱动。
- **主题自适应**：所有 UI 颜色、字体、间距、边框读取 `themeProvider` token。
- **来源可信度**：GameData > Wiki > Book；Book 引用必须附免责声明。
- **数据源边界**：不得继续把 `search_wiki` 作为 Agent 主知识源。
- **Git 规范**：按任务粒度小步提交，遵循 Conventional Commits。
